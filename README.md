# rebuilttheworld.org

**Public website and evolving system map for Rebuilt The World.**  
A structured gateway to archive, manifesto, Human & AI LAB, research paths, and long-term digital legacy.

---

## Overview

`rebuilttheworld.org` is the standalone repository for the public-facing website of **Rebuilt The World**.

This repository is not intended to be just a landing page project. It is being built as a **public gateway** to a wider ecosystem of ideas, documents, systems, research, and long-term legacy work.

The site is intended to function as:

- a **gateway**
- a **manifesto**
- a **system map**
- an **archive entry point**

---

## Core Direction

The current working definition is:

> **rebuilttheworld.org = gateway + manifesto + system map + archive entry point**

The goal is to create a public interface that communicates not only identity, but also structure, continuity, and meaning.

This repository is being developed with long-term readability in mind — for current collaborators, future readers, and the next generation.

---

## Project Layers

The project is being shaped across three connected layers:

### 1. Format
How the project looks, speaks, and feels.

Includes:
- visual language
- typography
- tone of voice
- interface rhythm
- brand consistency

### 2. System
How knowledge is structured, connected, and preserved.

Includes:
- information architecture
- module logic
- naming conventions
- documentation standards
- versioning
- archive logic

### 3. Website
How the public-facing experience is presented.

Includes:
- homepage structure
- navigation
- entry paths
- featured modules
- notes and signals
- archive access

## Intended Role of the Site

This website is being built as a structured public surface for the wider **Rebuilt The World** ecosystem.

It should eventually provide access to:

- the core mission and manifesto
- system-level thinking and project structure
- Human & AI LAB
- archive and research paths
- long-term documentation and legacy-oriented materials

---

## Proposed Site Structure

The site is expected to evolve around the following areas:

- **Home** — hero, mission, system map, entry paths
- **About** — what RTW is, why it exists, long-term vision
- **Manifesto** — principles, reconstruction logic, human–AI position
- **System** — modules, naming logic, documentation and versioning
- **LAB** — Human & AI LAB, methods, experiments, notes
- **Archive** — documents, field notes, releases, verification, timeline
- **Projects** — active and linked RTW modules
- **Signals** — short notes, fragments, updates, breadcrumbs
- **Entry** — paths for collaborators, future readers, and archive access

---

## Design Direction

The current design philosophy combines three directions:

### Archive Minimal
Clean typography, whitespace, document-like clarity, strong readability.

### Research / Lab Interface
Structured modules, visible systems, layered knowledge, living process.

### Mythic but Disciplined
A sense of weight, continuity, and legacy — without becoming vague or ornamental.

---

## Content Principles

Important pages should follow a consistent internal structure:

1. Title  
2. One-line definition  
3. Purpose  
4. Context  
5. Main content  
6. Related modules  
7. Version  
8. Archive references  

This helps the system remain understandable over time.
